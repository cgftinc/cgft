# creating the search environment

an environment defines the tools your model can use and the reward signals for training. for search/rag, this means a BM25 search tool and a reward function that checks if the model retrieved the right chunks.

## how it works

**the search tool** (`search_corpus`)
- gives the model BM25 search over your corpus
- takes a query string + optional metadata/filename filters
- returns ranked chunks with scores
- the model learns to write better queries + iteratively search for missing information via rl

**the reward function** (`compute_reward`)
- runs after the model generates an answer
- checks if the model's tool calls retrieved the reference chunks
- measures text overlap between retrieved content and ground truth chunks
- rewards ≥ 25% overlap, otherwise 0
- this teaches the model to search effectively, not just answer

during training, the model gets rewarded for using the search tool to find the right information before answering. over time it learns corpus-specific search patterns.

## using the built-in environment

`CorporaSearchEnv` is the built-in search environment for the CGFT corpus API. use it directly — no need to implement anything yourself:

```python
import synthetic_data_prep
from synthetic_data_prep.envs.corpora_search_env import CorporaSearchEnv
from synthetic_data_prep.trainer.pipeline import train

experiment_id = train(
    env_class=CorporaSearchEnv,
    env_args={
        "api_key": API_KEY,
        "corpus_id": source.corpus_id,
        "base_url": BASE_URL,
```

## custom environments

if you want to define your own tools and reward logic, see [../bring-your-env/your-own-env.mdx](../bring-your-env/your-own-env.mdx) for a full guide. you can subclass `SearchEnv` to reuse the shared reward function and dataset logic:

```python
from synthetic_data_prep.envs.search_env import SearchEnv
from benchmax.envs.types import ToolDefinition

class MySearchEnv(SearchEnv):
    def __init__(self, my_api_key: str, dataset_path: str, **kwargs):
        self._dataset_path = dataset_path

        search_tool = ToolDefinition(
            name="search_corpus",
            description="Search corpus using BM25",
            input_schema={
                "type": "object",
                "properties": {"query": {"type": "string"}},
                "required": ["query"],
            },
        )
        self._tools = {search_tool.name: (search_tool, self._search_corpus_tool)}

    async def _search_corpus_tool(self, query: str, **kwargs) -> str:
        # your search implementation here
        ...
```

see [notebooks/synthetic_rag_dataset_gen.ipynb](https://github.com/cgftinc/synthetic-data-prep-lib/blob/main/notebooks/synthetic_rag_dataset_gen.ipynb) for a full working example.
