# launching training

once you have a dataset and an environment, one call launches the full training pipeline.

## quickstart

```python
import synthetic_data_prep
from synthetic_data_prep.envs.corpora_search_env import CorporaSearchEnv
from synthetic_data_prep.trainer.pipeline import train

experiment_id = train(
    env_class=CorporaSearchEnv,
    env_args={
        "api_key": API_KEY,
        "corpus_id": source.corpus_id,
        "base_url": BASE_URL,
    },
    dataset=dataset.to_list(),
    api_key=API_KEY,
    local_modules=[synthetic_data_prep],
)
```

`train()` returns an experiment ID. view it at `https://app.cgft.io/experiments/{experiment_id}`.

see [notebooks/simple_rag_pipeline.ipynb](https://github.com/cgftinc/synthetic-data-prep-lib/blob/main/notebooks/simple_rag_pipeline.ipynb) for a full working example.

## parameters

```python
experiment_id = train(
    env_class=CorporaSearchEnv,       # your environment class
    env_args={...},                   # constructor args (api_key, corpus_id, etc.)
    dataset=list_of_dicts,
    api_key=API_KEY,
    local_modules=[synthetic_data_prep],
)
```

`dataset_path` is injected into `env_args` automatically — you don't need to pass it manually.

## monitoring training

### what to expect early on

rewards will fluctuate in the first few dozen steps — this is normal. the model is still unstable and learning basic patterns. give it time before drawing conclusions.

**pass@k vs average reward**: ideally pass@k climbs first. this means the model is learning to solve more distinct prompts (breadth) before squeezing out higher rewards on known prompts (depth). if average reward shoots up while pass@k stays flat, the model is overfitting to a narrow set of easy examples.

**healthy trajectory**: pass@k increases early → average reward follows with a lag → both plateau as the model saturates the training distribution.

### warning signs

- **pass@k flat, average reward increasing** — model is exploiting a narrow subset of prompts rather than building broad capability
- **both metrics stagnate early** — training distribution may be too hard or reward signal too sparse; try more single-hop examples or a simpler reward function
